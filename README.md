## 📋 Project Scope & Goal

The **FitTrack** project aims to design, develop, and publicly launch a cross-platform mobile fitness tracking application available on both iOS and Android. The app will allow users to log workouts, set personal fitness goals, monitor progress through data visualizations, and receive personalized recommendations. The project spans approximately five months — from initial planning through App Store and Google Play approval to full public release. Success is defined as a live, stable application with at least 500 downloads in the first two weeks post-launch, achieving a minimum 4.0-star rating across both app stores.

---

## 🏁 What Is a Milestone?

Based on the Atlassian article, a **milestone** is a significant checkpoint or achievement in a project's lifecycle — not an individual task. Milestones mark the *completion* of a phase or the delivery of a key outcome. For example:

- ✅ **Milestone:** "UI/UX Design Approved" ← marks the end of the design phase  
- ❌ **Task (not a milestone):** "Design the onboarding screen" ← a single action within the phase

A good milestone is:
- **Binary** — it's either done or not done
- **Meaningful** — it signals real progress to stakeholders
- **Time-bound** — it has a clear target date

## 🏷️ Status Legend

| Badge | Status | Meaning |
|-------|--------|---------|
| 🔵 Planned | Not Started | Milestone is scheduled but work has not begun |
| 🟡 In Progress | Active | Work toward this milestone is currently underway |
| 🟢 Complete | Done | Milestone has been achieved and verified |
| 🔴 At Risk | Blocked | Milestone may be delayed due to blockers or issues |
